- Pra enviar um repositório, ele precisa ter pelo menos 1 arquivo.

git status
    Verifica alterações e erros

git init
    Inicia um reposítorio na pasta pro git


git add <nome do arquivo incluindo formato>
git add .
    Adiciona o arquivo escolhido pro git
    Adiciona TODOS os arquivos da pasta pro git usando '.'.

git commit -m "first commit"
    Envia os arquivos adicionados ao git para o repositório. -m é uma flag de mensagem.

git branch -M main
git branch -M master
    Criação de um branch (ramificação) main/master. É onde o código correto estará depositado.

git remote add origin https://github.com/USUARIO/NOMEDOREPOSITORIO.git
    Linkagem e sincronização do repositório e preparo para envio para o github.

git push -u origin main
    Envio definitivo para o github


RESUMO:
    git init - inicia o git em uma pasta
    git add . - seleciona arquivos para preparo de envios pro git (intermediador)
    git commit -m "alteração" - envia pro git (intermediador)
    git branch -M main - Define o que foi enviado como raíz do repositório
    git remote add origin https://github.com/USUARIO/NOMEDOREPOSITORIO.git - Sync com o github
    git push -u origin main - Envio definitivo do git (intermediador) pro repositório do github (destinatário)

Envio:
    CÓDIGO NO PC > GIT (INTERMEDIADOR) > GITHUB
